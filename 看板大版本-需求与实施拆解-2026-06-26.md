## 0. 名词与数据映射(先对齐口径)

| 概念 | 对应字段/来源 | 说明 |
|---|---|---|
| **产品类** | registry `产品主线`(line) | 共 25 类(有领星数据)。如「移动空调」「保险箱」「电动猫砂盆」 |
| **型号/产品** | registry `product_name` = 领星 `local_name` | 如「移动空调」(VOTO)/「欧伦移动空调AO16TA」/「A018B移动空调」 |

## 3. 数据来源映射总表

| 看板元素 | 接口/快照字段 | 类聚合 | 齐全度 |
|---|---|---|---|
| 整体/类/型号盈亏 | compare(sales/profit/ad_cost) | 型号加总 | ✅ |

| 时间轴 | timeline_payout(按月) | 按 ym 合并 | ✅ |
| 采购/欠厂(投入) | procurement(finance_dim_contract) | 加总 | ✅ |
| 平台费用 | fees(佣金/FBA/广告/仓储/退款) | 加总 | ✅ |
| 广告 | ads(spend/acos) | 加总 | ✅ |
| 退款 | refund(率/量/额) | 加总 | ✅ |
| 品质星级 | quality_stars(by asin)+ registry.asins | 类下 asin 合并 | ✅ |
| 头程 | — | 估算 $15/台 | 🟡 估 |
| **运营费用/费用动作** | **opex(by 对方会计科目)** | 加总 | ✅ 6/26 通(测评/链接/品牌/差旅…) |

**opex 已打通(2026-06-26)**:finance-bot 接入金蝶 API + 常驻管线(LaunchAgent `ai.dokicat.finance-mirror-sync`,每天定时全量拉金蝶总账,40天窗口增量去重;今早回填 18,828 行,每晚 21:00 北京自动跑)。`对方一级科目` 是金蝶 API 自带字段,持续保鲜、非手工映射。opex 按 `对方会计科目`(category)出**费用动作**:测评费用(=刷单/测评)、链接费用、品牌商标、产品研发、运营差旅…
⚠️ caveat:① 归属靠「摘要 ILIKE 产品名」,偏漏;② 无报销人列 → 员工维度靠 registry 产品→负责人**间接**,不是 opex 直接带人。

## 5. 待确认 / 风险

1. ~~opex 待确认~~ **已通(6/26 finance-bot 接金蝶 + 常驻管线保鲜)**,见 §3。L2 运营费用卡、团队费用动作详情**可做**。残留:归属偏漏 + 无报销人维度(员工费用靠 registry 间接)。

4. **头程/运营** 是估算/占位,mockup L0 投入条要明确标 caveat,不当精确值。
5. **快照静态** — 数据非实时,认证+固定域名后切回实时 wx.request。
