# 运营 / 日报 数据字典 — 2026-06-28

> 两个本机 SQLite 源（不在领星/金蝶 DuckDB，走各自专用端点，都只读）。"日报 tab / 运营行动"用这两块。

---

## 1. 日报推送（outbox.db · 表 pushes）

- **库**：`duoqubao-bot/data/outbox.db`（SQLite，本机），macmini push 脚本 ssh 远程写入。
- **端点**：
  - 列表 `GET /feishu/reports?module=daily_report&hours=&user=&limit=`
  - 单条全文 `GET /feishu/reports/{id}` → `full_content`
- **表 pushes 字段**：`id · user_open_id · module · pushed_at · summary · full_content(JSON) · feishu_msg_id`
- **节奏**：每天约 07:22(UTC) 推；同一份日报会按用户多条（不同 user_open_id、同内容）。

### full_content（v4 结构，"日报 tab"主体）
```
date · version · data_source
USD_总销售额 · L7日均_USD · 回款_USD · 毛利润_USD · 广告_USD · FBM运费_USD
库龄危险: { msku_over_365 · qty_over_365 · msku_270plus · qty_270plus · top_items[[品名,数量]...] }
分段汇总: { A_主卖品 · B_ERP漏建 · C_种子链接 · D_base未维护 }（各含 组数/销售额_usd/占比）
按主负责运营: [{ owner · sales_usd · profit_usd · 品名数 }...]
raw_rows: [{ local_name · seller_name · asin · country · currency_code · qty · sales_amt · sales_usd ...}...]
```
> 已结构化、天天现成，渲染即可，不用再算。

## 2. 运营 checkin 行动（checkin_actions.sqlite · 表 checkin_actions）

- **库**：`~/.dokicat-ops/checkin_actions.sqlite`（SQLite，**duoqubao-api 同机本地**，非 macmini）。
- **写入**：dokicat-mini 的 review_worker **每天 4 次双写**（live，约 5369 行起）。
- **打开**：**只读 + WAL**（`file:...?mode=ro`，并发读安全，绝不写）。
- **端点**：`GET /checkin?since=YYYY-MM-DD&open_id=&limit=`（x-api-key 守，与 /deploy 同套鉴权）。

### 表 checkin_actions 字段（契约·客户 2026-06-28 给）
| 字段 | 类型 | 说明 |
|---|---|---|
| action_id | TEXT PK | 主键 |
| open_id | TEXT | 运营飞书 open_id |
| name | TEXT | 中文显示名（如 黄莹） |
| date_ms | INTEGER | 行动日期·epoch 毫秒 |
| category | TEXT | 8 类：广告调整 / 选品 / 客诉处理 / 上架优化 / 供应商谈判 / 库存调仓 / 对外协作 / 杂事其他 |
| summary | TEXT | ≤30 字摘要 |
| description | TEXT | 运营原话 |
| follow_up | TEXT | 待跟进 / 已闭环 / 不需要跟进 |
| product | TEXT | 中文品名（best-effort，可空） |
| source | TEXT | batch_extracted / bot_extracted |
| created_at | TEXT | 写入时间 ISO |

- 索引：`(open_id, date_ms)`
- 端点额外返回 `date`（date_ms → YYYY-MM-DD，方便用）。

### 样例（实测 2026-06-27）
```
林梦欣 | 客诉处理 | 测算损坏售后方案亏损      | 已闭环 |
林梦欣 | 对外协作 | 跟进遮阳伞case进度        | 待跟进 | 遮阳伞
刘晶玲 | 杂事/其他 | 重新提交两个链接SIPP审核 | 待跟进 |
```
> 用途：看每个运营每天干了啥（按 8 类）、待跟进状态、关联品名 → "日报 tab / 运营行动"视图。
